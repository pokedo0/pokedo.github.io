SS-client端搭建

 2018-04-06 2020-12-27 Tech 7 分钟读完 (大约1068个字)

仅为初用者撰写的基础教程

Windows

  1. 下载win客户端 https://github.com/shadowsocks/shadowsocks-windows/releases
    相应的混淆插件 https://github.com/shadowsocks/simple-obfs/releases

  2. 下载并解压后，将obfs文件夹内的所有文件复制到shadowsocks所在文件夹下，如图.

  3. 进入网站，点击左侧节点列表，再点击节点查看配置信息，同时提供二维码扫码方式.

  4. 如图打开ss后，再右键ss菜单，点击 扫描屏幕的二维码 方式导入配置信息.
    如果扫码失败，可以手动填写信息.

    注意ss软件中的插件和插件选项按如下填写:
    插件 : obfs-local
    插件选项 : obfs=http;obfs-host=www.bing.com

  5. 接下来右键ss菜单，点击 启用系统代理 ，再在 系统代理模式 中切换到 全局模式。
    打开edge浏览器，显示 无法连接到代理服务器 的报错， 转到代理设置中，按下图填写，再保存。
    切换到edge再次浏览网站，即可成功登录Google.

    如果在这步不成功，请跳到第7.步骤

  6. 因为是全局模式，所以访问国内网站会经过代理服务器，速度会变得很慢，所以我们需要切换到pac模式。
    首先右键选择pac栏，选择 从gfwlist更新本地pac ，显示更新完成的提示后，再点击pac模式。
    切回浏览器上测试登录Google， youtube， 可以在百度上看到ip是本地ip， 同时也可以上youtube网站。

  7. 以上涉及的代理都是全局模式，所以理论上系统所有的socks5流量都会经过ss处理，如果单单要浏览网页的话，就显得大材小用，处理效率不高。
    因此我们需要通过一个外置的插件，通过设置端口监听ss的本地端口，使得ss只处理浏览器的流量。
    SwitchyOmega便可以实现这个功能。其官网上已经有完备的教程了，我简单贴下链接和描述教程。
    下载： https://www.switchyomega.com/download.html
    配置： https://www.switchyomega.com/settings.html

    白话文 ( Chrome为示例 )：

    下载后缀为.crx的拓展，并拖到chomre里面安装。
    右键保存 为.bak的文件，并在Omega中选择 从备份文件中恢复 导入代理列表。
    点击左侧 自动切换 栏，在右侧点击 立即更新情景模式 ，用来更新最新的代理规则。
    返回chrome的浏览界面，点击右上角对应的Omega拓展图标，选择 自动切换 模式，同时，右键ss，取消启用系统代理 的选项。(下图例)
    在chrome中登录Google测试，可以正常登录，在edge中则无法登录Google。

Android

  1. 下载APP https://github.com/shadowsocks/shadowsocks-android/releases
    相应的混淆插件 https://github.com/shadowsocks/simple-obfs-android/releases

  2. 打开ss，点击右上角扫描网站提供的二维码。

  3. 编辑节点信息，只修改功能设置这一块，路由设置为GFW列表。同时插件栏一定要选中Simple obfuscation。

  4. 返回配置文件界面，点击右下角的图标开始连接到服务器，再点击测试连接，如果成功出现延时xx毫秒字样，则表示连接成功。
